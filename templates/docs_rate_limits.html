Rate Limit Headers

Every API response includes rate limit information in the headers to help you track your usage.

Response Headers Example 

HTTP/1.1 200 OK
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 245
X-RateLimit-Reset: 1694234567
X-RateLimit-Retry-After: 45
X-RateLimit-Window: 60
Content-Type: application/json

X-RateLimit-Limit

The maximum number of requests allowed in the current window.

300 requests per minute

X-RateLimit-Remaining

The number of requests remaining in the current window.

245 requests remaining

X-RateLimit-Reset

Unix timestamp when the current window will reset.

1694234567 reset timestamp

X-RateLimit-Retry-After

Seconds to wait before making another request (when rate limited).

45 seconds to wait

X-RateLimit-Window

The size of the rate limit window in seconds.

60 second window

Best Practices

Follow these best practices to optimize your API usage and avoid rate limit issues.

Implement Exponential Backoff

When you receive a 429 error, wait for an increasing amount of time between retries. 

def exponential_backoff(attempt):
    return min(300, (2 ** attempt) + random())

Monitor Rate Limit Headers

Always check the rate limit headers to proactively manage your request rate. 

remaining = response.headers['X-RateLimit-Remaining']
if int(remaining) < 10:
    time.sleep(60)

Batch Similar Requests

Group similar requests together to reduce the total number of API calls. 

POST /api/v1/agents/batch
{
  "requests": [request1, request2, request3]
}

Cache Responses

Cache API responses when appropriate to reduce redundant requests. 

if cache.has(request_key):
    return cache.get(request_key)
cache.set(request_key, api_response, ttl=300)

Use Streaming for Long Responses

For long-running tasks, use streaming endpoints to avoid timeouts. 

response = requests.post(url, json=data, 
                        stream=True, 
                        headers={'Accept': 'text/event-stream'})
