API Key Authentication

API keys are the simplest way to authenticate your requests. Include your API key in the Authorization header of every request.

Header Authentication

Include your API key in the Authorization header using the Bearer token format:

HTTP Request 
Authorization: Bearer sk-1234567890abcdef1234567890abcdef

cURL Example

curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://api.intelcoreai.com/v1/chat

Python Example

headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}
requests.get(url, headers=headers)

Security Warning

Never expose your API keys in client-side code, public repositories, or logs. Keep them secure on your server.

OAuth 2.0 Authentication

OAuth 2.0 provides secure, token-based authentication ideal for applications that need to act on behalf of users.

Authorization Code Flow

1

Authorization Request

Redirect user to authorization server

→
2

User Consent

User grants permissions

→
3

Authorization Code

Receive code via callback

→
4

Access Token

Exchange code for token

Step 1: Authorization URL

Authorization Request 
https://auth.intelcoreai.com/oauth/authorize?
  response_type=code&
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://yourapp.com/callback&
  scope=read write&
  state=random_state_value

Step 2: Token Exchange

Token Request 
curl -X POST https://auth.intelcoreai.com/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code" \
  -d "code=AUTHORIZATION_CODE" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "redirect_uri=https://yourapp.com/callback"

Available Scopes

read

Read access to user data and AI responses

write

Create and modify conversations and settings

admin

Full administrative access (enterprise only)

Security Best Practices

Follow these security guidelines to protect your API keys and user data.

Key Management

  • Rotate API keys regularly
  • Use different keys for different environments
  • Store keys in environment variables
  • Never commit keys to version control

Network Security

  • Always use HTTPS in production
  • Validate SSL certificates
  • Implement IP whitelisting
  • Use VPN for sensitive operations

Token Lifecycle

  • Set appropriate token expiration
  • Implement token refresh mechanisms
  • Revoke tokens on logout
  • Monitor token usage patterns

Attack Prevention

  • Implement rate limiting
  • Monitor for unusual activity
  • Use CSRF protection
  • Validate all input data
